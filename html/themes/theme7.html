Тема: Алгоритмы с ветвлениями

    В языке Паскаль используется два оператора для реализации условных переходов - IF и CASE, а также оператор безусловного перехода GOTO(этот оператор не рассматривается). Они позволяют нарушить последовательный порядок выполнения инструкций программы.



Оператор условного перехода

Оператор условного перехода в PascalABC имеет вид:
1.    if <условие>
2.       then <оператор 1>
3.       else <оператор 2>;

Условие - это логическое выражение, в зависимости от которого выбирается одна из двух альтернативных ветвей алгоритма.
Если значение условия истинно (TRUE), то будет выполняться оператор 1, записанный после ключевого слова then.
В противном случае будет выполнен оператор 2, следующий за словом else, при этом оператор 1 пропускается.
После выполнения указанных операторов программа переходит к выполеннию команды, стоящей непосредственно после оператора if.

Перед ключевым словом else точка с запятой НЕ СТАВИТСЯ!
Если поставить точку с запятой, то PascalABC будет воспринимать (;) как конец логического ветвления.

else - часть в операторе if может отсутствовать:
1.    if <условие> then <оператор 1>

Тогда в случае невыполнения логического условия управление сразу передается оператору, стоящему в программе после конструкции if.

Синтаксис языка допускает запись только одного оператора после ключевых слов then и else, поэтому группу инструкций обязательно надо объединять в составной оператор(окаймлять операторными скобками begin...end).
Иначе возникает(чаще всего) логическая ошибка программы, когда компилятор языка ошибок не выдает, но программа тем не менее работает неправильно.
1.    if (x>y)
2.       then writeln('true')
3.       else writeln('false');

1.    if (n<=10) then max+=n;   //max:=max+n;

1.    if (x=0) then begin
2.       min:=x;
3.       n+=1;
4.    end;



Оператор выбора

Часто возникают ситуации, когда приходится осуществлять выбор одного из нескольких альтернативных путей выполнения программы. Несмотря на то, что такой выбор можно организовать с помощью оператора if...then, удобнее воспользоваться специальным оператором выбора. Его формат:
1.    case <выражение> of
2.       вариант(1): <оператор 1>;
3.       вариант(2): <оператор 2>;
4.       ...
5.       вариант(n): <оператор n>;
6.    end;

Или с оператором else:
1.    case <выражение> of
2.       вариант(1): <оператор 1>;
3.       вариант(2): <оператор 2>;
4.       ...
5.       вариант(n): <оператор n>;
6.       else <оператор>;
7.    end;

Выражение, которое записывается после ключевого слова case, называется селектором, оно может быть любого перечисляемого типа.

Вариант состоит из одной или большего количества констант или диапазонов, разделенных запятыми.

Они должны принадлежать к тому же типу, что и селектор, причем недопустимо более одного упоминания вариантав записи инструкции case. Из перечисленного множества операторов будет выбран только тот, перед которым записан вариант, совпадающий со значением селектора. Если такого варианта нет, выполняется оператор, следующий за словом else(если он есть).

1.    case s of
2.       'A'..'Z', 'a'..'z': writeln('Строка');
3.       '0'..'9': writeln('Число');
4.       '+', '-', '*', '/': writeln('Знак');
5.       else writeln('Спец. символ');
6.    end;
